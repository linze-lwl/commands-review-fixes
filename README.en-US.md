# Alibaba Cloud Function Compute Documentation

[简体中文](README.md)

Welcome to the Alibaba Cloud Function Compute documentation repository.

This repository is used to organize and collaboratively maintain Function Compute documentation. The content is based on publicly available Alibaba Cloud Function Compute documentation and is reorganized by product area for review, contribution, and long-term maintenance.

## Documentation Scope

Chinese documentation is located in `docs/zh-CN/` and organized by product area:

- 01.Function
- 02.FC Agent Sandbox
- 03.AgentRun
- 04.FunModel
- 05.FunArt

English documentation is located in `docs/en-US/` and will be added progressively by product area.

Each product area follows a consistent structure where applicable:

- 01.Getting Started
- 02.Release Notes
- 03.Pricing
- 04.Features
- 05.Best Practices
- 06.Security Compliance
- 07.Developer Reference
- 08.FAQ
- 09.Support

Cloud Sandbox developer reference includes E2B SDK compatibility APIs, connection parameters, and troubleshooting guidance. Platform-generated API Reference pages are not maintained manually in this repository.

## Contributing

Documentation contributions include not only text changes, but also issue reports, missing topics, invalid links, outdated screenshots, incorrect examples, and structure problems.

### Submit Issues

Internal and external users are welcome to submit Issues for documentation problems or improvement suggestions. Please include:

- The affected documentation path or online URL;
- The problem found;
- The expected fix or improvement;
- Screenshots or reproduction steps when helpful.

Maintainers will review Issues regularly and convert actionable items into documentation updates.

### Submit Pull Requests

Developers are welcome to submit Pull Requests for fixes, additions, and improvements. Before submitting a PR, please make sure:

- The change scope is clear;
- The content is accurate and supported by a clear source;
- Images or media referenced by Markdown files are committed together;
- Platform-generated developer reference content is not edited manually;
- The original document structure and visual intent are preserved where applicable.

All PRs require maintainer review and PTL Review before merge.

## Repository Structure

- `docs/zh-CN/`: Chinese Markdown product documentation.
- `docs/en-US/`: English Markdown product documentation.
- `assets/zh-CN/`: Images and media referenced by Chinese documentation.
- `assets/en-US/`: Images and media referenced by English documentation.
- `README.md`: Chinese README.
- `README.en-US.md`: English README.
