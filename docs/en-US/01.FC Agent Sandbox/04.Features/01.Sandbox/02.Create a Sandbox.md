# Create a Sandbox

Creating a Sandbox is the first step in using FC Agent Sandbox. After the Sandbox is ready, you can run commands, read and write files, execute code, or start temporary HTTP services.

## Create a Default Sandbox

```typescript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create({
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});

try {
  const result = await sandbox.commands.run("echo hello");
  console.log(result.stdout.trim());
} finally {
  await sandbox.kill();
}
```

## Specify a Template

When you need preinstalled dependencies or a fixed runtime environment, create the Sandbox from a template:

```typescript
const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  timeoutMs: 10 * 60 * 1000,
});
```

Common `Sandbox.create()` parameters include:

- `template`: Template name or template ID.
- `timeoutMs`: Sandbox timeout in milliseconds.
- `envs`: Environment variables injected into the Sandbox runtime.
- `metadata`: Custom metadata written to the Sandbox control plane.

## Recommendations

After creating the Sandbox, your application code should call `sandbox.kill()` in a `try/finally` block. If a task needs to preserve its current state, use [Pause and Resume](06.Pause%20and%20Resume.md), but do not use pause as a substitute for releasing resources.
