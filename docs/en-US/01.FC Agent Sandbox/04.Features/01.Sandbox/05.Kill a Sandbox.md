# Kill a Sandbox

Killing a Sandbox releases its resources. Explicitly kill the Sandbox when a task finishes, encounters an unrecoverable error, or exceeds the time your application is willing to wait.

## Kill the Current Sandbox

```typescript
await sandbox.kill();
```

Release resources in `finally`:

```typescript
const sandbox = await Sandbox.create({
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});

try {
  await sandbox.commands.run("npm test");
} finally {
  await sandbox.kill();
}
```

## Difference from pause

`kill()` ends the Sandbox lifecycle, while `pause()` suspends the Sandbox so it can be resumed later. After a Sandbox is killed, you cannot return to the same runtime state with `Sandbox.connect()`.

If the task only needs to wait briefly for the user to continue, use [Pause and Resume](06.Pause%20and%20Resume.md). If the task is complete, kill the Sandbox.
