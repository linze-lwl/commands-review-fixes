# Dynamic OSS Mounts

Dynamic OSS mounts let you mount an OSS path into a sandbox. After the mount is complete, the sandbox can read from or write to the mounted directory like a local file system path.

This feature must be configured in the Function Compute console, the FC Agent Sandbox control plane, or related cloud products first. Application code only needs to use the agreed mount path.

## Configuration entry and effective scope

Before integration, confirm the OSS mount configuration entry and effective scope:

- Configuration entry: Use the currently available FC Agent Sandbox console or control plane capability as the source of truth. If the console does not show a storage mount configuration entry for FC Agent Sandbox, contact product support to confirm the enablement method.
- Effective scope: Confirm whether the mount configuration applies to the account, region, template, sandbox, or session, and whether it affects only newly created sandboxes.
- Mount parameters: Confirm the OSS bucket, object prefix, mount path inside the sandbox, and read/write permissions.

After the cloud-side configuration is complete, application code only needs to read and write files through the agreed path. Storage type, permission model, and performance may vary by enablement method. Validate with production-scale data before production integration.

## Read OSS files

```python
from e2b import Sandbox
import os

sandbox = Sandbox.create(
    "code-interpreter-v1",
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)

try:
    mount_path = os.environ.get("OSS_MOUNT_PATH", "/mnt/oss")
    input_file = f"{mount_path}/inputs/data.csv"

    result = sandbox.commands.run(
        f"python3 - <<'PY'\n"
        f"from pathlib import Path\n"
        f"path = Path({input_file!r})\n"
        f"print(path.exists())\n"
        f"print(path.stat().st_size if path.exists() else 0)\n"
        f"PY"
    )

    print(result.stdout.strip())
finally:
    sandbox.kill()
```

## Write OSS files

```typescript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});

try {
  const outputPath = `${process.env.OSS_MOUNT_PATH ?? "/mnt/oss"}/outputs/result.txt`;

  const result = await sandbox.commands.run(
    `python3 - <<'PY'
import os
from pathlib import Path

path = Path(os.environ["OUTPUT_PATH"])
path.parent.mkdir(parents=True, exist_ok=True)
path.write_text("sandbox task finished\\n", encoding="utf-8")
print(path)
PY`,
    {
      envs: {
        OUTPUT_PATH: outputPath,
      },
    }
  );

  console.log(result.stdout.trim());
} finally {
  await sandbox.kill();
}
```

## Common configuration items

- OSS bucket: The bucket to be mounted.
- OSS path: The directory or object prefix to mount.
- Mount path: The local path inside the sandbox used to access OSS, for example `/mnt/oss`.
- Access mode: Read-only or read-write.

## Notes

- Do not expose an entire bucket as writable to untrusted code.
- In multi-tenant scenarios, use isolated directory prefixes such as `tenants/<tenant-id>/tasks/<task-id>/`.
- Configure OSS lifecycle cleanup rules for temporary artifacts when appropriate.

For configuration details, see the official documentation: [Dynamic OSS Mounts](https://help.aliyun.com/zh/functioncompute/mount-oss-dynamically).
