# Overview

Templates define the base runtime environment for a sandbox. By preinstalling system dependencies, language runtimes, toolchains, and application code in a template, you can reduce the initialization work required after each sandbox is created.

## When to use templates

- You need fixed versions of Python, Node.js, system packages, or command-line tools.
- You need to preinstall the dependencies required for an agent task.
- You want to move repetitive post-startup installation steps into the build phase.
- You need to reuse the same runtime environment across multiple sandboxes.

## Basic workflow

1. Prepare a base image or use a built-in template.
2. Build the template with `Template.build()`.
3. Save the template ID or template name.
4. Specify the template when creating a sandbox.

```typescript
const sandbox = await Sandbox.create("<template-id-or-name>", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```
