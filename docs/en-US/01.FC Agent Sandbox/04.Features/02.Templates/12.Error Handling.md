# Error Handling

Template build failures usually come from the image, network, dependencies, resource specifications, or permissions. The goal is to identify the failing stage quickly and preserve enough context for retries or manual troubleshooting.

## Catch build errors

```python
import os

from e2b import Template, TemplateError

try:
    build = Template.build(
        Template().from_image(os.environ["FROM_IMAGE"]),
        name="agent-runtime",
        api_key=os.environ["E2B_API_KEY"],
        api_url=os.environ["E2B_API_URL"],
        domain=os.environ["E2B_DOMAIN"],
    )
except TemplateError as error:
    print("template build failed:", error)
    raise
```

TypeScript example:

```typescript
import { Template, TemplateError } from "e2b";

try {
  const build = await Template.build(
    Template().fromImage(process.env.FROM_IMAGE),
    "agent-runtime",
    {
      apiKey: process.env.E2B_API_KEY,
      apiUrl: process.env.E2B_API_URL,
      domain: process.env.E2B_DOMAIN,
    },
  );
} catch (error) {
  if (error instanceof TemplateError) {
    console.error("template build failed:", error);
  }
  throw error;
}
```

## Troubleshooting order

- Check whether the image address uses the VPC internal domain.
- Check whether the registry, VPC, and vSwitch are in the same region.
- Check whether ACR EE permissions and security groups allow access.
- Check whether the base image includes the tools required for the build.
- Check whether the template resource specifications are sufficient for dependency installation and startup.
