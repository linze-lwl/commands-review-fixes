# Start and Ready Commands

E2B templates support startup commands and readiness checks in a template so that a sandbox can automatically start a service and wait until the service becomes available after creation.

FC Agent Sandbox should currently prefer starting application processes by running commands after the sandbox is created:

```typescript
const server = await sandbox.commands.run("python3 -m http.server 8000", {
  background: true,
});

const host = sandbox.getHost(8000);
console.log(`https://${host}`);
```

## Recommendations

If the service startup step is required every time a sandbox is created, install the dependencies in the template first, then start the service with a background command after sandbox creation. Validate template-level startup and readiness commands against the actual SDK behavior and build results before relying on them.
