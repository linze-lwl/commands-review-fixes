# Define a Template

Defining a template describes the steps used to build a sandbox runtime environment. FC Agent Sandbox currently submits template builds through `Template.build()` in the E2B SDK.

## Define a template from an image

```python
from e2b import Template

template = Template().from_image("<acr-ee-vpc-domain>/<namespace>/<image>:<tag>")
```

TypeScript example:

```typescript
import { Template } from "e2b";

const template = Template().fromImage("<acr-ee-vpc-domain>/<namespace>/<image>:<tag>");
```

## Submit the build

```python
import os

from e2b import Template

build = Template.build(
    template,
    name="agent-runtime",
    cpu_count=2,
    memory_mb=2048,
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)
```

TypeScript example:

```typescript
const build = await Template.build(template, "agent-runtime", {
  cpuCount: 2,
  memoryMB: 2048,
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```

## Recommendations

The template definition should include only runtime environment content. Write task input, user data, and temporary state after the sandbox is created to avoid coupling templates with application data.
