# E2B Compatibility and Migration

## How can an existing E2B application connect to FC Agent Sandbox?

Keep the E2B SDK invocation pattern unchanged first, and replace only the API key, API URL, and domain.

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.<region>.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="<region>.e2b.fc.aliyuncs.com"
```

After the change, verify four things first: sandbox creation, command execution, file reads and writes, and resource release. Do not start by migrating complex templates, network logic, or storage logic.

## Why do official E2B examples not configure an endpoint?

Official E2B examples target E2B services by default. FC Agent Sandbox provides an E2B-compatible endpoint on the Function Compute side, so `E2B_API_URL` and `E2B_DOMAIN` must be configured explicitly.

`E2B_API_URL`, `E2B_DOMAIN`, the template, and the sandbox must all belong to the same region. If the regions do not match, common results include creation failures, connection failures, or templates not being found.

## Does the E2B CLI still need `E2B_ACCESS_TOKEN`?

Do not keep using `E2B_ACCESS_TOKEN` in new integration flows. It is a deprecated legacy E2B authentication variable. New E2B CLI versions should use `E2B_API_KEY`.

```bash
export E2B_API_KEY="<your-api-key>"
```

If the SDK works but the CLI does not, check the CLI version first, and then verify `E2B_API_KEY`, `E2B_API_URL`, and `E2B_DOMAIN`.

## Is FC Agent Sandbox exactly the same as E2B?

No. FC Agent Sandbox first provides an E2B SDK-friendly developer experience so that existing E2B applications can migrate to FC Agent Sandbox with relatively low refactoring cost. On top of that, FC Agent Sandbox continues to extend enterprise capabilities on Alibaba Cloud infrastructure.

From a product-positioning perspective, FC Agent Sandbox is not a simple copy of E2B. It preserves an E2B-friendly developer experience while enhancing cloud runtime capabilities, such as VPC connectivity, enterprise-grade fine-grained access control, connectivity to cloud resources such as NAS, OSS, and RDS, and governance and integration capabilities for enterprise production environments. Over time, FC Agent Sandbox will be closer to a capability superset of "E2B-friendly experience plus enterprise cloud enhancements."

Migration still needs to be evaluated against currently available capabilities. For Snapshot, Volume, Team, access token management, logs, Network Config Update, high-frequency metrics, or cloud-enhanced capabilities, first check [Getting Started / E2B Compatibility](../01.Getting%20Started/05.E2B%20Compatibility.md), or contact the product or sales team to confirm the current supported scope, enablement method, and usage boundaries.

## Can API keys be managed through the E2B SDK?

No. FC Agent Sandbox API keys are created and managed in the Function Compute console, not through the E2B SDK.

In production, isolate API keys by application, environment, or workspace. Do not write full API keys into code, templates, logs, command output, downloaded files, or frontend pages.

## Why do Python and TypeScript examples use different parameter names?

This is a language-style difference across SDKs. Common Python parameters include `api_key` and `api_url`, while common TypeScript parameters include `apiKey` and `apiUrl`.

When troubleshooting, follow the type definitions and examples for the current language SDK. Do not copy field names directly from another language.

## What if sandbox creation succeeds after migration, but `run_code` or port access fails?

These issues are usually not authentication issues. They are more commonly caused by templates, startup commands, service readiness, or port listening.

First validate `run_code`, command execution, and file reads and writes with a built-in template. If the built-in template works, focus the custom template check on Code Interpreter dependencies, startup commands, listening ports, and readiness conditions.

## Can E2B Volume, Storage, and Snapshot be migrated directly?

Do not assume so by default. FC Agent Sandbox should currently be evaluated against the capabilities that are already compatible.

If the original application relies on Volume or Snapshot to preserve state, confirm first whether FC Agent Sandbox currently supports the corresponding interfaces. If it relies on Storage, Archil, cloud buckets, or FUSE mounts, confirm first whether they are part of the formal compatibility path. If support cannot be confirmed, do not present them in the FAQ as recommended solutions.
