# Use AIO Sandbox

AIO Sandbox is designed for Agent tasks where browser interaction, code execution, and file processing happen in sequence. If you already have an `aio` template that includes a browser, Playwright or Puppeteer, Python or Node.js, and common data-processing dependencies, you can complete page access, screenshots, downloads, data cleanup, code execution, and report generation inside the same sandbox. This reduces data movement between the browser environment and the code execution environment.

This practice fits post-scraping analysis, report generation after automated testing, content production Agents, operations console inspection, and multi-tool Agents. Compared with Browser Use Sandbox alone, AIO Sandbox is better suited to tasks where browser artifacts must be handed off directly to Python or Node.js scripts for further processing.

## Use cases

- Collect data from a dynamic website and immediately run a Python script to clean, aggregate, and export it.
- Complete browser actions and then parse, transcode, or turn downloaded files into reports.
- Run browser E2E tasks and backend scripts together in automated testing.
- Generate screenshots, PDFs, spreadsheets, and archived files for content production Agents.

## Recommended workflow

1. Create FC Agent Sandbox from a prepared `aio` template.
2. In the browser stage, access pages, interact, take screenshots, or download files.
3. In the code stage, read browser artifacts and complete parsing, cleanup, calculation, and report generation.
4. In the audit stage, save the URL, action summary, screenshot paths, script version, and output results.
5. In the cleanup stage, release the sandbox and remove task-level credentials and temporary files.

## Example code

The following example assumes the `aio` template already includes Playwright, a browser, and Python. It first collects information from a web page and then generates an analysis result.

```javascript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("aio", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  envs: { TASK_TYPE: "aio-browser-code" },
  metadata: { scenario: "aio-browser-code" },
  timeoutMs: 300_000,
});

try {
  await sandbox.files.makeDir("/tmp/aio-task");
  await sandbox.files.write(
    "/tmp/aio-task/collect.mjs",
    `import fs from "node:fs/promises";
import { chromium } from "playwright";

const browser = await chromium.launch({ headless: true });
const page = await browser.newPage({ viewport: { width: 1365, height: 768 } });
await page.goto("https://example.com", { waitUntil: "networkidle" });
await page.screenshot({ path: "page.png", fullPage: true });

const rows = await page.evaluate(() => [{
  url: location.href,
  title: document.title,
  textLength: document.body.innerText.length,
}]);

await fs.writeFile("browser-result.json", JSON.stringify(rows, null, 2));
await browser.close();
`,
  );
  await sandbox.files.write(
    "/tmp/aio-task/analyze.py",
    `import json
from pathlib import Path

rows = json.loads(Path("browser-result.json").read_text())
summary = {
    "page_count": len(rows),
    "titles": [row["title"] for row in rows],
    "total_text_length": sum(row["textLength"] for row in rows),
    "artifacts": ["page.png", "browser-result.json"],
}
Path("summary.json").write_text(json.dumps(summary, ensure_ascii=False, indent=2))
print(json.dumps(summary, ensure_ascii=False))
`,
  );

  const collect = await sandbox.commands.run("node collect.mjs", {
    cwd: "/tmp/aio-task",
    timeoutMs: 120_000,
  });
  if (collect.exitCode !== 0) {
    throw new Error(collect.stderr || collect.stdout);
  }

  const analyze = await sandbox.commands.run("python3 analyze.py", {
    cwd: "/tmp/aio-task",
    timeoutMs: 60_000,
  });
  if (analyze.exitCode !== 0) {
    throw new Error(analyze.stderr || analyze.stdout);
  }

  console.log(JSON.parse(analyze.stdout));
} finally {
  await sandbox.kill();
}
```

## Production recommendations

- Record the browser stage and code stage separately, with independent inputs, outputs, logs, and errors.
- Validate the type, size, and path of downloaded files before passing them to the code stage.
- Both page content and downloaded files may carry prompt injection. Do not treat raw page text as a system instruction.
- Inject browser accounts, cookies, model keys, and business credentials at runtime and destroy them when the task ends.
- Set timeouts, file size limits, and page-count limits for expensive tasks to prevent unbounded crawling or computation.
