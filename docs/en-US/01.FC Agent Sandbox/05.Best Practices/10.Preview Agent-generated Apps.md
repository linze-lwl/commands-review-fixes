# Preview Agent-generated Apps

App Builders, Coding Agents, and automated testing systems often need to start a temporary web service so a user or upstream system can inspect the generated result. The value of preview is not just getting a URL. It is putting code generation, service startup, health checks, and result confirmation into the same isolated environment.

FC Agent Sandbox works well as a temporary preview environment. An Agent can write generated frontend pages, backend services, or static files into the sandbox, start a service with a background command, and confirm availability through a local health check inside the sandbox. If the preview must be exposed externally, you can obtain an access address through `getHost(port)` and then verify accessibility according to the current access-control policy.

## Use cases

- An App Builder generates a page and immediately starts a preview service for user confirmation.
- A Coding Agent modifies frontend code and returns a temporary access address with build logs.
- An automated test starts a web service inside the sandbox and then runs API checks or page smoke checks.
- When multiple users generate apps concurrently, each task gets its own isolated preview environment.

## Solution design

Split the preview flow into four steps:

1. Write the generated artifacts, such as `index.html`, application code, or build output.
2. Start the service with a background command and record the process ID.
3. Run a health check inside the sandbox to confirm that the service is listening and returning the expected content.
4. Return the preview address or check result, then stop the background process and release the sandbox when the task ends.

If the preview address will be exposed directly to end users, evaluate access control, sensitive file exposure, service lifetime, and debug interface risks separately.

## Example code

The following example starts a static page service inside the sandbox and performs a local in-sandbox health check first.

```javascript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  envs: { TASK_TYPE: "web-preview" },
});

let server;

try {
  await sandbox.files.makeDir("/tmp/web-preview");
  await sandbox.files.write(
    "/tmp/web-preview/index.html",
    `<!doctype html>
<html>
  <head><title>Agent Preview</title></head>
  <body><h1>Agent generated preview</h1></body>
</html>
`,
  );

  server = await sandbox.commands.run("python3 -m http.server 8000 --bind 127.0.0.1", {
    cwd: "/tmp/web-preview",
    background: true,
  });

  const check = await sandbox.commands.run(
    `python3 - <<'PY'
import time
import urllib.request

last_error = None
for _ in range(30):
    try:
        body = urllib.request.urlopen("http://127.0.0.1:8000", timeout=2).read().decode()
        assert "Agent generated preview" in body
        print("preview_ok")
        break
    except Exception as exc:
        last_error = exc
        time.sleep(1)
else:
    raise RuntimeError(last_error)
PY`,
    { timeoutMs: 30_000 },
  );
  if (check.exitCode !== 0) {
    throw new Error(check.stderr || check.stdout);
  }

  console.log({
    localCheck: check.stdout.trim(),
    previewHost: sandbox.getHost(8000),
  });
} finally {
  if (server?.pid) {
    await sandbox.commands.kill(server.pid);
  }
  await sandbox.kill();
}
```

## Production recommendations

- Run the in-sandbox health check before handing the preview address to the user.
- Before exposing the preview publicly, confirm the directory does not contain credentials, logs, environment files, or debug endpoints.
- Treat preview services as task-scoped and short-lived. They are not recommended as long-running production services.
- If public access returns a permission error, troubleshoot against the current FC Agent Sandbox access-control policy instead of bypassing authentication in the business system.
- Preinstall frontend build dependencies, package managers, and browser test dependencies in custom templates where possible.

## Runnable example

This practice maps to `npm run web-preview` in the sample project. The script starts a local preview service inside FC Agent Sandbox, completes the health check, and outputs the address returned by `getHost(port)`.
