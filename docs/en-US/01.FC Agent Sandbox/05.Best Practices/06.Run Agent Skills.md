# Run Agent Skills

Skills are a good way to turn complex Agent capabilities into reusable task packages. A Skill can include instructions, an entry script, dependency definitions, test samples, and output format constraints. Compared with prompts alone, Skills are easier to version, audit, and regression test, and they are better suited for reuse inside an enterprise.

FC Agent Sandbox works well as the execution environment for Skills. The business system chooses a Skill for the task, writes the Skill package and input files into the sandbox, runs the entry script in an isolated environment, and then reads back the structured result. This avoids letting Skill dependencies pollute the main service and gives every execution a clear input, output, and lifecycle.

## Use cases

- Standardized tasks such as data quality checks, data cleanup, and report generation.
- Engineering tasks such as repository analysis, test execution, and dependency inspection.
- Office automation tasks such as document processing, format conversion, and summary generation.
- Enterprise teams packaging fixed diagnostic workflows as Skills that multiple Agents can reuse.

## Skill package structure

A maintainable Skill should include at least:

```text
data-quality/
  SKILL.md
  run.py
  tests/
  output_schema.json
```

`SKILL.md` describes usage conditions, inputs, and outputs. The entry script performs the work. Test cases and the output schema ensure the Skill stays regressible and verifiable. In production, Skill packages should live in a source repository or artifact repository and be referenced by version.

## Example code

The following example writes a `data-quality` Skill into FC Agent Sandbox, checks a CSV file for missing values and duplicate order IDs, and returns a business recommendation.

```javascript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  envs: { TASK_TYPE: "agent-skill" },
});

try {
  await sandbox.files.makeDir("/tmp/agent-skills");
  await sandbox.files.makeDir("/tmp/agent-skills/skills");
  await sandbox.files.makeDir("/tmp/agent-skills/skills/data-quality");

  await sandbox.files.write(
    "/tmp/agent-skills/skills/data-quality/run.py",
    `import csv, json, sys
from collections import Counter

input_path = sys.argv[1]
rows = list(csv.DictReader(open(input_path, newline="")))
missing_cells = sum(1 for row in rows for value in row.values() if value == "")
order_ids = [row["order_id"] for row in rows]
duplicates = [order_id for order_id, count in Counter(order_ids).items() if count > 1]
recommendation = "needs_cleanup_before_analysis" if missing_cells or duplicates else "ready_for_analysis"
print(json.dumps({
    "row_count": len(rows),
    "missing_cells": missing_cells,
    "duplicate_order_ids": duplicates,
    "recommendation": recommendation
}, ensure_ascii=False))
`,
  );
  await sandbox.files.write(
    "/tmp/agent-skills/input.csv",
    "order_id,customer,amount,status\n1001,Acme,1200,paid\n1002,Acme,,paid\n1002,Acme,900,paid\n",
  );

  const result = await sandbox.commands.run("python3 skills/data-quality/run.py input.csv", {
    cwd: "/tmp/agent-skills",
    timeoutMs: 30_000,
  });
  if (result.exitCode !== 0) {
    throw new Error(result.stderr || result.stdout);
  }

  console.log(JSON.parse(result.stdout));
} finally {
  await sandbox.kill();
}
```

## Production recommendations

- Version every Skill package. Keep the Skill name, version, input summary, and output summary in execution records.
- Expose one stable entry point per Skill, such as `run.py`, `main.ts`, or `entrypoint.sh`.
- Use JSON for outputs and validate fields and types with a schema.
- Put common dependencies into custom templates to avoid installing them during every run.
- High-risk Skills should have permission tiers and approvals, especially for internal system access, write operations, or sensitive data handling.

## Runnable example

This practice maps to `npm run skills` in the sample project. The script writes the Skill package and input data into FC Agent Sandbox, runs it, and returns a data quality conclusion.
