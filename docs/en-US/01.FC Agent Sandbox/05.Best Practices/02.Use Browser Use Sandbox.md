# Use Browser Use Sandbox

Browser Use Sandbox is designed for Agents that need to open web pages, click buttons, fill forms, archive screenshots, or scrape dynamic pages. If you already have a `browser` template that includes a browser, fonts, and Playwright or Puppeteer dependencies, you can create a browser sandbox directly through the E2B SDK and run web actions inside an isolated, disposable, and auditable environment.

This practice fits web data collection, operations console automation, screenshot archiving, page inspection, lightweight end-to-end testing, and browser tool Agents. The Agent decides which page to visit, what actions to perform, and how to interpret the result. Browser Use Sandbox is responsible for running the browser, saving screenshots and downloaded files, and limiting the sandbox lifecycle.

## Use cases

- Scrape data from a dynamic website and then pass it to a code executor for cleanup and aggregation.
- Automatically fill web forms, click buttons, download files, and generate screenshots or PDFs.
- Give marketing, customer service, or operations Agents the ability to observe and interact with web pages.
- Run browser E2E tests inside an isolated environment without polluting CI workers or business services.

## Recommended workflow

1. Create FC Agent Sandbox from a prepared `browser` template.
2. Write the task URL, action parameters, and any short-lived credentials into the sandbox.
3. Run a Playwright or Puppeteer script inside the sandbox.
4. Output structured results and save screenshots, HTML, PDFs, or downloaded files into the task directory.
5. Read the results and release the sandbox.

## Example code

The following example assumes the `browser` template already includes Playwright and browser dependencies. It opens a web page, extracts the title and a text summary, and saves a screenshot.

```javascript
import { Sandbox } from "e2b";

const sandbox = await Sandbox.create("browser", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
  envs: { TASK_TYPE: "browser-use" },
  metadata: { scenario: "browser-use" },
  timeoutMs: 300_000,
});

try {
  await sandbox.files.makeDir("/tmp/browser-task");
  await sandbox.files.write(
    "/tmp/browser-task/task.json",
    JSON.stringify({
      url: "https://example.com",
      screenshotPath: "/tmp/browser-task/page.png",
    }),
  );

  await sandbox.files.write(
    "/tmp/browser-task/run-browser-task.mjs",
    `import fs from "node:fs/promises";
import { chromium } from "playwright";

const task = JSON.parse(await fs.readFile("task.json", "utf8"));
const browser = await chromium.launch({ headless: true });
const page = await browser.newPage({ viewport: { width: 1365, height: 768 } });

await page.goto(task.url, { waitUntil: "networkidle", timeout: 60_000 });
await page.screenshot({ path: task.screenshotPath, fullPage: true });

const result = await page.evaluate(() => ({
  title: document.title,
  text: document.body.innerText.replace(/\\s+/g, " ").slice(0, 500),
}));

await browser.close();
console.log(JSON.stringify(result));
`,
  );

  const result = await sandbox.commands.run("node run-browser-task.mjs", {
    cwd: "/tmp/browser-task",
    timeoutMs: 120_000,
  });

  if (result.exitCode !== 0) {
    throw new Error(result.stderr || result.stdout);
  }

  console.log(JSON.parse(result.stdout));
} finally {
  await sandbox.kill();
}
```

## Production recommendations

- Treat browser tasks as handling untrusted web pages by default, and restrict allowed domains, downloadable file types, and output size.
- Page content may contain prompt injection. Do not let the Agent execute instructions from the page without validation.
- Isolate login state, cookies, and account credentials per task, inject them at runtime, and do not bake them into the template.
- Record the URL, screenshots, key DOM summaries, tool-call trace, and result files for every external page interaction.
- If a task needs to continue with Python or Node.js processing after browser actions, prefer [Use AIO Sandbox](04.Use%20AIO%20Sandbox.md).
