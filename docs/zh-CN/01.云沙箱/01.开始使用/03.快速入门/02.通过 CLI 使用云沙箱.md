# 通过 CLI 使用云沙箱

本文引导你通过 E2B CLI 查看模板、创建 Sandbox、进入终端并清理资源。

## 前置条件

- 已拥有阿里云账号，并具备函数计算相关操作权限。
- 已在函数计算控制台[创建 API Key](../02.创建%20API%20Key.md)。
- 已确认目标地域支持云沙箱，并且本机网络可访问该地域的云沙箱服务。
- 已安装满足版本要求的 E2B CLI。版本要求参见[使用约束](../04.使用约束.md)。

## 1. 安装 CLI

macOS 可以使用 Homebrew：

```bash
brew install e2b
```

也可以使用 npm：

```bash
npm i -g @e2b/cli
```

## 2. 配置环境变量

配置 API Key、API URL 和域名。API Key 的创建和管理方式参见[创建 API Key](../02.创建%20API%20Key.md)。

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.<region>.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="<region>.e2b.fc.aliyuncs.com"
```

其中 `<region>` 为云沙箱服务所在地域。支持地域和 Endpoint 规则参见[使用约束](../04.使用约束.md)。

`E2B_ACCESS_TOKEN` 是 E2B 已废弃的旧认证变量。新版本 E2B CLI 应使用 `E2B_API_KEY`。如果旧版 CLI 仍要求 `E2B_ACCESS_TOKEN`，请先升级 CLI 后重试。

可以先列出 Sandbox 验证配置是否生效：

```bash
e2b sandbox list
```

## 3. 查看模板

```bash
e2b template list
```

输出中可以看到当前账号可用的模板。以下只展示输出格式，模板 ID、规格、创建时间和状态会随账号、地域和模板配置变化：

```text
Access  Template ID    Template Name         vCPUs  RAM MiB  Created at    Disk size MiB  names                buildStatus
Public  <template-id>  code-interpreter-v1       2     2048  <created-at>          10240  code-interpreter-v1  ready
```

## 4. 创建并连接 Sandbox

```bash
e2b sandbox create code-interpreter-v1
```

进入 Sandbox 终端后，可以执行：

```bash
python3 -c 'print("hello from sandbox")'
exit
```

## 5. 连接已有 Sandbox

如果已有运行中的 Sandbox，可以通过 Sandbox ID 重新连接交互式终端：

```bash
e2b sandbox connect <sandbox-id>
```

退出终端只会关闭本地终端会话，不会自动终止该 Sandbox。

## 6. 在 Sandbox 中执行命令

也可以不进入交互式终端，直接在已有 Sandbox 中执行命令：

```bash
e2b sandbox exec <sandbox-id> ls
```

命令输出会返回到本地终端。

## 7. 查看 Sandbox 指标

查看运行中 Sandbox 的 CPU、内存和磁盘指标：

```bash
e2b sandbox metrics <sandbox-id>
```

当前指标按 1 分钟粒度返回，适合调试和观察资源使用趋势。

## 8. 查看和终止 Sandbox

查看 Sandbox 列表：

```bash
e2b sandbox list
```

终止不再使用的 Sandbox：

```bash
e2b sandbox kill <sandbox-id>
```

任务完成后建议主动终止 Sandbox，避免资源持续占用。
