# 产品简介

云沙箱（FC Agent Sandbox）是面向 AI Agent 和代码执行场景的云端隔离运行环境。开发者可以按需创建一个 Sandbox，在其中执行命令、运行代码、读写文件、启动临时服务，并在任务完成后释放资源。

云沙箱适合承载不应直接运行在业务服务进程内的任务，例如 AI 生成代码执行、数据分析、自动化脚本、依赖复杂的工具调用和临时 Web 服务。应用侧负责创建和编排 Sandbox，Sandbox 负责提供独立、可释放的执行环境。

如果你已经熟悉 E2B，可以继续使用 E2B SDK 或 E2B CLI 接入云沙箱；如果你是第一次使用，建议先[创建 API Key](02.创建%20API%20Key.md)，再从[通过 SDK 使用云沙箱](03.快速入门/01.通过%20SDK%20使用云沙箱.md)开始。

## 你可以用云沙箱做什么

- **运行 Agent 工具**：为 Agent 提供独立执行环境，运行命令、处理文件、调用工具链。
- **构建代码解释器**：执行 Python、Shell 等代码，返回 stdout、stderr、文本结果或文件产物。
- **处理临时数据任务**：上传数据文件，在 Sandbox 内完成清洗、转换、分析或生成报告。
- **启动临时服务**：在 Sandbox 内启动 HTTP 服务、开发服务器或调试服务，并通过端口访问地址调用。
- **固化运行环境**：通过模板预装依赖、语言运行时和工具链，减少每次任务的初始化成本。

## 核心对象

### Sandbox

Sandbox 是一次远端隔离执行环境。应用创建 Sandbox 后，可以在其中运行命令、处理文件、执行代码或启动服务。任务完成后，应主动终止 Sandbox，避免资源持续占用。

### Template

Template 定义 Sandbox 启动时的运行环境。模板中可以固化基础镜像、语言运行时、依赖包和工具链。对启动速度、依赖一致性或工具版本有要求时，建议使用自定义模板。

### Commands

Commands 用于在 Sandbox 中执行命令或进程。常见用法包括运行脚本、启动后台服务、执行测试、连接运行中进程和发送标准输入。

### Filesystem

Filesystem 用于管理 Sandbox 内的文件。应用可以写入输入文件、读取结果文件、管理目录、重命名文件或判断路径是否存在。

### Code Interpreter

Code Interpreter 用于在 Sandbox 中执行代码，适合数据分析、代码解释器和 Agent 代码执行场景。常用模板为 `code-interpreter-v1`。

### Network

Network 能力用于访问 Sandbox 暴露的端口。需要对外访问 Sandbox 内服务时，可以获取指定端口的访问地址。

### Storage

Sandbox 本地文件系统适合当前任务内使用。需要长期保存或跨任务共享的数据，应使用 NAS、OSS 等外部存储方案。

### FC Extensions

FC Extensions（云上扩展）用于将云沙箱接入阿里云网络、存储、日志和监控体系，例如 VPC 网络配置、动态挂载 OSS、日志采集和监控告警。这些能力是云沙箱（FC Agent Sandbox）相对 E2B 的云上增强，不属于 E2B SDK 原生对象。

## 基本使用路径

1. 选择模板，例如 `code-interpreter-v1` 或自定义模板。
2. 创建 Sandbox。
3. 写入输入文件、环境变量或任务参数。
4. 执行命令、运行代码或启动服务。
5. 读取 stdout、stderr、文件结果或端口访问地址。
6. 任务完成后终止 Sandbox。
