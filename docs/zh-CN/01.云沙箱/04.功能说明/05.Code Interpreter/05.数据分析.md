# 数据分析

Code Interpreter 常见用法是把数据写入 Sandbox，再通过 `run_code()` 执行分析、转换或可视化代码。该模式适合构建数据分析、报表生成和 AI 代码执行类应用。

## 写入 CSV 并执行分析

Python 示例：

```python
import os
import textwrap

from e2b_code_interpreter import Sandbox

sandbox = Sandbox.create(
    template="code-interpreter-v1",
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)

try:
    sandbox.files.write(
        "/tmp/sales.csv",
        "month,revenue\n2026-01,120\n2026-02,180\n2026-03,160\n",
    )

    execution = sandbox.run_code(
        textwrap.dedent(
            """
            import pandas as pd

            df = pd.read_csv("/tmp/sales.csv")
            print(df)
            df["revenue"].sum()
            """
        )
    )

    stdout = "".join(execution.logs.stdout or [])
    stderr = "".join(execution.logs.stderr or [])

    print("stdout:")
    print(stdout.strip())
    print("stderr:")
    print(stderr.strip())
    print("text:")
    print(execution.text)

    if execution.error:
        raise RuntimeError(execution.error)
finally:
    sandbox.kill()
```

## 输出结果说明

- `execution.logs.stdout`：`print()` 输出。
- `execution.logs.stderr`：标准错误输出。
- `execution.text`：最后一个裸表达式的文本结果。
- `execution.error`：代码执行错误信息。

## 使用建议

- 小数据文件可以用 `sandbox.files.write()` 写入；大文件建议使用上传地址。
- 分析代码应显式读取 Sandbox 内路径，例如 `/tmp/input.csv`。
- 对用户上传的数据和 AI 生成的代码，应设置超时、文件大小限制和输出大小限制。
- 任务结束后主动调用 `sandbox.kill()`，避免资源持续占用。
