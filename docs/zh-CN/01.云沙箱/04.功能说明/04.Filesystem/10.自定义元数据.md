# 自定义元数据

E2B Filesystem 支持在写入文件时附加自定义键值元数据。元数据会随文件保存，并可通过 `getInfo()`、`list()` 和 `rename()` 返回，适合记录文件来源、任务 ID、内容类型或业务标签。

## 写入元数据

Python SDK 版本如果支持 `metadata` 参数，可以按以下方式写入：

```python
info = sandbox.files.write(
    "/tmp/report.txt",
    "hello\n",
    metadata={
        "author": "alice",
        "purpose": "demo",
    },
)

print(info.metadata)
```

TypeScript 示例：

```typescript
const info = await sandbox.files.write("/tmp/report.txt", "hello\n", {
  metadata: {
    author: "alice",
    purpose: "demo",
  },
});

console.log(info.metadata);
```

批量写入多个文件时，传入的同一组元数据会应用到本次写入的所有文件。

Python 示例：

```python
sandbox.files.write(
    "/tmp/input/a.txt",
    "A\n",
    metadata={
        "source": "import",
    },
)
sandbox.files.write(
    "/tmp/input/b.txt",
    "B\n",
    metadata={
        "source": "import",
    },
)
```

TypeScript 示例：

```typescript
await sandbox.files.write(
  [
    { path: "/tmp/input/a.txt", data: "A\n" },
    { path: "/tmp/input/b.txt", data: "B\n" },
  ],
  {
    metadata: {
      source: "import",
    },
  },
);
```

## 读取元数据

Python 示例：

```python
info = sandbox.files.get_info("/tmp/report.txt")
print(info.metadata)

entries = sandbox.files.list("/tmp")
for entry in entries:
    print(entry.name, entry.metadata)
```

TypeScript 示例：

```typescript
const info = await sandbox.files.getInfo("/tmp/report.txt");
console.log(info.metadata);

const entries = await sandbox.files.list("/tmp");
for (const entry of entries) {
  console.log(entry.name, entry.metadata);
}
```

## 限制

- 元数据键和值应使用可打印 ASCII 字符。
- 键名会被沙箱转换为小写，读取时大小写可能与写入时不同。
- 覆盖写入文件会替换原有元数据，不会保留上一次写入的元数据。
- 该能力依赖运行环境支持文件扩展属性。E2B 官方要求模板内 envd 版本为 `v0.6.2` 或以上；使用较早构建的自定义模板时，应先验证模板内运行时版本是否支持该能力。
- Python SDK 的 `metadata` 参数需要较新版本支持；如果当前 SDK 不支持，请升级 SDK 或通过 TypeScript SDK 写入元数据。

## 使用建议

元数据适合保存轻量业务标签，不适合作为权限控制、审计日志或持久化数据库。需要长期保存、检索或跨 Sandbox 共享的业务状态，应写入外部存储。
