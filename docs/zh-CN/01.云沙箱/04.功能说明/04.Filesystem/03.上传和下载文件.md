# 上传和下载文件

上传和下载地址适合处理大文件、浏览器直传或结果文件下载。小文本文件和 Agent 生成代码优先使用 `sandbox.files.write()` 和 `sandbox.files.read()`。

## 生成下载地址

当 Sandbox 内已经生成文件时，可以通过 `sandbox.downloadUrl(path)` 获取下载地址，再由业务侧把文件交给用户或后续系统处理。

```typescript
await sandbox.files.write("/tmp/result.txt", "analysis result\n");

const downloadUrl = sandbox.downloadUrl("/tmp/result.txt");
console.log(downloadUrl);
```

## 生成上传地址

需要把大文件上传到 Sandbox 时，可以先通过 `sandbox.uploadUrl(path)` 获取上传地址，再从业务侧上传到指定路径。

```typescript
const uploadUrl = sandbox.uploadUrl("/tmp/input.csv");
console.log(uploadUrl);
```

上传完成后，可以在 Sandbox 内直接读取或处理该文件：

```typescript
const result = await sandbox.commands.run("python3 - <<'PY'\nfrom pathlib import Path\nprint(Path('/tmp/input.csv').exists())\nPY");
console.log(result.stdout.trim());
```

## 注意事项

- `downloadUrl(path)` 和 `uploadUrl(path)` 生成的是指定路径的访问地址，使用前要确认路径符合业务预期。
- 不要把包含敏感数据的上传、下载地址写入日志或暴露给无关用户。
- 大文件上传后建议通过 `sandbox.files.exists()` 或命令检查文件是否可读。
- 文件只在当前 Sandbox 生命周期内可用。需要长期保存时，应写入外部存储。

## 选择建议

- 小文件、文本文件、Agent 生成代码：使用 `sandbox.files.write()` 和 `sandbox.files.read()`。
- 大文件、浏览器直传、下载结果文件：使用 `sandbox.uploadUrl()` 和 `sandbox.downloadUrl()`。
- 目录遍历、重命名或删除：使用 Filesystem API。
