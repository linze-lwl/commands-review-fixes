# 概览

Filesystem 用于管理沙箱内文件和目录。Agent 常用它写入输入文件、生成代码、读取测试结果、上传大文件和下载产物。

## 常用能力

- `sandbox.files.read()` / `sandbox.files.write()`：读写文件。
- `sandbox.files.list()`：列出目录内容。
- `sandbox.files.exists()`：判断路径是否存在。
- `sandbox.files.getInfo()`：获取文件或目录元信息。
- `sandbox.files.makeDir()`：创建目录。
- `sandbox.files.remove()`：删除文件或目录。
- `sandbox.files.rename()`：移动或重命名。
- `sandbox.files.write(path, data, { metadata })`：写入文件时附加[自定义元数据](10.自定义元数据.md)。
- `sandbox.uploadUrl()` / `sandbox.downloadUrl()`：生成上传或下载 URL。

## 推荐路径

小文件、二进制内容和流式内容优先使用 Filesystem API。需要让浏览器等未持有 SDK 鉴权信息的环境上传或下载文件时，使用上传下载 URL，并为 URL 设置合理的过期时间。需要跨沙箱保存的数据应写入外部持久化存储。

目录监听可用于等待 Agent 输出文件或同步任务产物，参见[监听变化](11.监听变化.md)。
