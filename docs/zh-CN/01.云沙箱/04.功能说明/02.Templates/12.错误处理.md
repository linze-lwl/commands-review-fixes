# 错误处理

模板构建失败通常来自镜像、网络、依赖、资源规格或权限问题。错误处理的目标是快速定位失败阶段，并保留足够上下文供重试或人工排查。

## 捕获构建错误

```python
import os

from e2b import Template, TemplateError

try:
    build = Template.build(
        Template().from_image(os.environ["FROM_IMAGE"]),
        name="agent-runtime",
        api_key=os.environ["E2B_API_KEY"],
        api_url=os.environ["E2B_API_URL"],
        domain=os.environ["E2B_DOMAIN"],
    )
except TemplateError as error:
    print("template build failed:", error)
    raise
```

TypeScript 示例：

```typescript
import { Template, TemplateError } from "e2b";

try {
  const build = await Template.build(
    Template().fromImage(process.env.FROM_IMAGE),
    "agent-runtime",
    {
      apiKey: process.env.E2B_API_KEY,
      apiUrl: process.env.E2B_API_URL,
      domain: process.env.E2B_DOMAIN,
    },
  );
} catch (error) {
  if (error instanceof TemplateError) {
    console.error("template build failed:", error);
  }
  throw error;
}
```

## 排查顺序

- 镜像地址是否使用 VPC 内网域名。
- 镜像仓库、VPC、vSwitch 是否同地域。
- ACR EE 权限和安全组是否允许访问。
- 基础镜像是否包含构建所需工具。
- 模板资源规格是否满足依赖安装和启动需求。
