# 自定义模板

系统预置模板提供了基础运行环境。当需要预装业务依赖、配置特定工具链或固化运行环境时，可以创建自定义模板。模板构建完成后，创建 Sandbox 时可以直接使用该模板，避免每次重复安装依赖。

## 准备 ACR EE 镜像

使用自有镜像创建模板时，需要在与云沙箱相同 UID、相同地域下准备 ACR EE 实例，并为 ACR EE 实例添加专有网络，确保云沙箱可以通过 VPC 拉取镜像。

将镜像推送到该 ACR EE 仓库，得到支持 VPC 访问的内网镜像地址，例如：

```text
<acr-ee-vpc-domain>/<namespace>/<image>:<tag>
```

## 安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install e2b==2.26.0 e2b-code-interpreter python-dotenv
```

## 配置环境变量

创建 `.env` 文件：

```bash
E2B_API_KEY="<your-api-key>"
FROM_IMAGE="<acr-ee-vpc-domain>/<namespace>/<image>:<tag>"

E2B_API_URL="https://api.<region>.e2b.fc.aliyuncs.com"
E2B_DOMAIN="<region>.e2b.fc.aliyuncs.com"
```

## 构建 Template 并验证

```python
#!/usr/bin/env python3

import os
import time

from dotenv import load_dotenv
from e2b import Template, default_build_logger
from e2b_code_interpreter import Sandbox


load_dotenv()

API_KEY = os.getenv("E2B_API_KEY", "").strip()
FROM_IMAGE = os.getenv("FROM_IMAGE", "").strip()
API_URL = os.getenv("E2B_API_URL", "").strip()
DOMAIN = os.getenv("E2B_DOMAIN", "").strip()
RUN_CODE = "print('hello')"
OPTS = {"api_key": API_KEY, "api_url": API_URL, "domain": DOMAIN}

build = Template.build(
    Template().from_image(FROM_IMAGE),
    name=f"template-{int(time.time())}",
    cpu_count=2,
    memory_mb=2048,
    skip_cache=False,
    on_build_logs=default_build_logger(),
    **OPTS,
)

sandbox = Sandbox.create(template=build.template_id, timeout=900, **OPTS)

try:
    print(f"template_id: {build.template_id}")
    print(f"build_id: {build.build_id}")
    print(f"sandbox_id: {sandbox.sandbox_id}")

    execution = sandbox.run_code(RUN_CODE, timeout=60, request_timeout=120)
    stdout = "".join(execution.logs.stdout or [])
    stderr = "".join(execution.logs.stderr or [])

    print(f"run_code stdout: {stdout.strip()}")
    print(f"run_code stderr: {stderr.strip()}")
    print(f"run_code error: {execution.error}")

    if execution.error is not None:
        raise RuntimeError(f"run_code 执行失败: {execution.error}")
finally:
    sandbox.kill()
```

## ACR EE 网络要求

- ACR EE 实例已至少绑定一个专有网络 VPC。
- 该 VPC 下至少有一个 vSwitch 位于函数计算支持的可用区。
- ACR EE 镜像仓库、VPC、vSwitch 与云沙箱位于同一地域。
- ACR EE 访问控制与 VPC 配置允许对应网络访问。
- VPC 网段使用 RFC 1918 规定的私有地址段。
- VPC 下至少配置一个非云服务托管的安全组，且安全组规则允许访问对应的 ACR EE 实例。

## 模板名称建议

建议在模板名称中包含业务名、基础镜像、关键依赖版本或日期，例如 `agent-python313-20260531`。生产环境避免覆盖正在使用的模板，建议先创建新模板并灰度验证。
