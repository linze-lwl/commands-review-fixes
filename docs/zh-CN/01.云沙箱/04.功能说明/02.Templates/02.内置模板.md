# 内置模板

内置模板由云沙箱平台提供和维护，用于快速创建标准 Sandbox。当前内置模板覆盖基础命令与文件访问、代码执行、浏览器自动化，以及代码执行和浏览器协同的复合场景。

如果您只是需要 Python、Node.js、文件管理、代码执行或浏览器自动化等通用能力，建议优先使用内置模板。只有当内置模板无法满足业务依赖、系统库、运行时版本或企业标准化要求时，再创建自定义镜像模板。

## 模板列表

| 模板 | 适用场景 | 说明 |
| --- | --- | --- |
| [base 模板](02.内置模板/base-模板.md) | 基础命令、文件访问、SDK 连通性验证 | 默认基础模板，不提供 Code Interpreter 服务或浏览器自动化能力 |
| [code-interpreter-v1 模板](02.内置模板/code-interpreter-v1-模板.md) | AI Agent 代码执行、数据分析、文件处理 | 快速入门和代码解释器示例默认使用 `code-interpreter-v1` |
| [browser 模板](02.内置模板/browser-模板.md) | 浏览器自动化、网页抓取、截图、UI 测试 | 提供 CDP、VNC、截图、页面交互等浏览器能力 |
| [all in one 模板](02.内置模板/all-in-one-模板.md) | 代码执行和浏览器自动化协同 | 同时提供 Code Interpreter 能力和 browser 能力 |

## 查看模板

使用 CLI 查看当前账号可用模板：

```bash
export E2B_API_KEY="<your-api-key>"
export E2B_API_URL="https://api.<region>.e2b.fc.aliyuncs.com"
export E2B_DOMAIN="<region>.e2b.fc.aliyuncs.com"

e2b template list
```

## 使用模板创建 Sandbox

使用 `e2b_code_interpreter` SDK 时，默认创建 `code-interpreter-v1` 沙箱，通常不需要显式指定模板。

使用通用 `e2b` SDK 时，可通过 `template` 参数显式选择模板。Python 示例：

```python
import os

sandbox = Sandbox.create(
    template="code-interpreter-v1",
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)
```

TypeScript 示例：

```typescript
const sandbox = await Sandbox.create("code-interpreter-v1", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```
