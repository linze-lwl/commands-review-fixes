# 工作原理

模板把沙箱启动前需要准备的运行环境固化下来。开发者不需要在每次创建沙箱后重新安装依赖，而是把这些动作前移到模板构建阶段。

## 模板和沙箱的关系

- 模板定义基础环境。
- 沙箱是基于模板创建出来的运行实例。
- 多个沙箱可以使用同一个模板。
- 更新模板不会自动修改已经运行中的沙箱。

## 适合放进模板的内容

- 系统依赖和语言运行时。
- Python、Node.js 等包管理器依赖。
- 常用 CLI 工具。
- 业务 Agent 的基础代码和配置。

## 不适合放进模板的内容

- 用户输入数据。
- 单次任务的临时文件。
- 会频繁变化的业务状态。
- 密钥、Token 和其他敏感凭证。
