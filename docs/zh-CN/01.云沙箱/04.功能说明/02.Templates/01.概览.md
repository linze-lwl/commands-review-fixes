# 概览

模板用于定义沙箱的基础运行环境。通过模板预装系统依赖、语言运行时、工具链和业务代码，可以减少每次创建沙箱后的初始化工作。

## 什么时候使用模板

- 需要固定 Python、Node.js、系统包或命令行工具版本。
- 需要预装 Agent 执行任务所需的依赖。
- 需要把冷启动后的重复安装步骤前置到构建阶段。
- 需要在多个沙箱之间复用相同运行环境。

## 基本流程

1. 准备基础镜像或使用内置模板。
2. 使用 `Template.build()` 构建模板。
3. 保存模板 ID 或模板名称。
4. 创建沙箱时指定模板。

```typescript
const sandbox = await Sandbox.create("<template-id-or-name>", {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```

Python 示例：

```python
sandbox = Sandbox.create(
    template="<template-id-or-name>",
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)
```
