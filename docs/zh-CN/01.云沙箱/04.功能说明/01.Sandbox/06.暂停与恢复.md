# 暂停与恢复

暂停用于在一段时间内保留 Sandbox 状态，后续通过连接同一个 Sandbox 继续使用。

## 暂停 Sandbox

暂停当前 Sandbox：

```python
sandbox.pause()
```

TypeScript 示例：

```typescript
await sandbox.pause();
```

暂停后，Sandbox 不再处于活跃运行状态。后续可以通过连接已有 Sandbox 的方式恢复使用。

## 连接时自动恢复

`Sandbox.connect(sandboxId)` 支持连接已有 Sandbox。如果 Sandbox 已暂停，连接时会自动恢复。

```python
import os

from e2b_code_interpreter import Sandbox

sandbox = Sandbox.connect(
    sandbox_id,
    api_key=os.environ["E2B_API_KEY"],
    api_url=os.environ["E2B_API_URL"],
    domain=os.environ["E2B_DOMAIN"],
)
```

TypeScript 示例：

```typescript
const sandbox = await Sandbox.connect(sandboxId, {
  apiKey: process.env.E2B_API_KEY,
  apiUrl: process.env.E2B_API_URL,
  domain: process.env.E2B_DOMAIN,
});
```

## 使用建议

- 对需要短期保留上下文的任务，可以在任务间隔期间暂停 Sandbox。
- 恢复后需要重新确认长连接、进程状态和业务层状态是否符合预期。
- 暂停不等于终止。任务完成后，应调用 `sandbox.kill()` 释放资源。
