# 「不兼容」概览

E2B 的 Agents 章节是一组 Agent 集成教程，主要围绕预构建模板、Agent CLI、Agent 自身 API Key、仓库操作和命令执行展开。它不同于 E2B SDK 的基础 Sandbox、Commands、Filesystem、Templates 等能力。

云沙箱当前不兼容 E2B Agents 章节中的预构建 Agent 模板和专属集成教程，不应在官方功能说明中作为正式能力引导开发者使用。

## 不兼容范围

- E2B 预构建 Agent 模板，例如 `amp`、`claude-code`、`codex`、`opencode` 等。
- 基于这些模板的 CLI 启动命令。
- Agent 专属线程管理、JSON streaming、权限自动批准等集成教程。
- E2B Agent 页面中的示例模板继承方式，例如从 `amp`、`claude-code` 等模板继续构建。

## 可以复用的底层能力

云沙箱仍可以作为 Agent 运行环境使用。开发者可以基于已兼容的能力自行构建 Agent：

- 使用 [Templates](../02.Templates/01.概览.md) 预装 Agent 所需依赖。
- 使用 [Commands](../03.Commands/01.概览.md) 启动 Agent CLI 或工具进程。
- 使用 [Filesystem](../04.Filesystem/01.概览.md) 写入代码、配置和任务文件。
- 使用 [Sandbox](../01.Sandbox/01.生命周期.md) 管理运行生命周期。

在云沙箱正式提供对应预构建模板前，文档不应直接照搬 E2B Agents 页面中的模板名称和调用示例。
