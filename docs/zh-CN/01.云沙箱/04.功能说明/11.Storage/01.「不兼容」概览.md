# 「不兼容」概览

E2B 的 Storage 章节用于把外部存储挂载到沙箱中，当前包括 Archil 和 Cloud buckets。它不同于 E2B Volume API：Storage 更偏向在自定义模板中安装第三方 FUSE 或挂载工具，并在沙箱运行时把对象存储或外部文件系统挂载为 POSIX 目录。

云沙箱当前不兼容 E2B Storage 文档中的挂载方案，不应在官方功能说明中作为正式能力引导开发者使用。

## 不兼容范围

- Archil 挂载。
- Google Cloud Storage FUSE 挂载。
- Amazon S3 `s3fs` 挂载。
- Cloudflare R2 `s3fs` 挂载。
- 依赖 FUSE、第三方挂载命令或外部存储凭证在沙箱内完成运行时挂载的方案。

## 替代方向

需要持久化或共享数据时，建议优先使用云上标准存储能力，并由业务系统显式管理读写、权限和生命周期，例如 OSS、NAS、数据库或业务已有存储服务。沙箱内临时文件读写使用 [Filesystem](../04.Filesystem/01.概览.md)，E2B Volume API 边界参见 [Volumes](../12.Volumes/01.「不兼容」概览.md)。
