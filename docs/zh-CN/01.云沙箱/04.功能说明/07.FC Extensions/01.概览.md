# FC Extensions 概览

FC Extensions（云上扩展）用于将云沙箱（FC Agent Sandbox）接入函数计算和阿里云基础设施能力，包括 VPC 网络配置、自定义域名、动态挂载 OSS、日志采集和监控日志。

FC Extensions 不改变 E2B SDK / CLI 的基本使用方式。应用仍然使用 E2B SDK 创建 Sandbox、执行命令、读写文件；VPC、自定义域名、OSS、日志和监控等能力通过函数计算控制台、控制面配置或阿里云相关服务配置生效。

## 功能列表

- [VPC 网络配置](02.VPC%20网络配置.md)：让 Sandbox 访问 VPC 内的数据库、内网 API、镜像仓库或其他云资源。
- [自定义域名](03.自定义域名.md)：使用固定域名和自有证书访问云沙箱 API 及沙箱内服务。
- [动态挂载 OSS](04.动态挂载%20OSS.md)：将 OSS 路径挂载到 Sandbox 内，按本地文件路径读取或写入对象。
- [监控与日志](05.监控与日志.md)：查看 Sandbox 运行状态、资源指标和日志，并将运行日志采集到日志服务。

## 使用方式

1. 在函数计算或相关云产品中完成扩展能力配置。
2. 使用 E2B SDK / CLI 创建 Sandbox。
3. 在 Sandbox 中访问对应网络、路径或输出日志。
4. 在控制台、日志服务或监控页面查看结果。

## 注意事项

- FC Extensions 不是 E2B SDK 新增方法，不需要修改 SDK 导入方式。
- VPC、自定义域名、OSS、日志和监控能力需要先完成云上配置，代码示例只展示应用侧如何使用和验证。
- 不要在模板、环境变量、日志或挂载路径中暴露长期凭证。
