# 用户界面功能详解

业务开发者通过开发者控制台（/user/）创建和使用 AI Agent。开发者控制台提供广场、历史、我的 Agent、知识库、记忆与数据库、工具与技能等核心功能，以及养虾 FunClaw、养马 FunHermes 等专题入口。

## 广场

广场是 Agent 的发现页面，供开发者探索和发现优质的 Agent、模型、工具与知识库。广场包含以下四个子 Tab：

| Tab | 说明 |
| --- | --- |
| Agent | 展示可用的 Agent 模板和示例，开发者可基于模板快速创建 Agent 实例。 |
| 模型 | 浏览平台上可用的 AI 模型。 |
| 工具 | 发现可用的工具和技能。 |
| 知识库 | 浏览可用的知识库资源。 |

## 历史

查看会话历史记录，包含 Session ID、统计信息和时间等。基于 OpenTelemetry 提供可观测数据，帮助了解 Agent 的运行状况。

## 我的 Agent

管理已创建的 AgentRuntime，支持创建对话型和流程编排型两类智能体。

支持以下操作：

- 创建新的 AgentRuntime。
- 复制已有的 AgentRuntime。
- 删除不再需要的 AgentRuntime。
- 查看 Agent 的访问地址和运行时端点。

Agent 实例以卡片网格形式展示，支持按名称或描述搜索、查看当月 Token 使用情况。首次进入无实例时，系统自动引导创建。

### 创建 Agent 实例

在**我的 Agent**页面中单击创建按钮，按以下流程创建 Agent 实例。

#### 选择智能体类型

首先选择要创建的智能体类型：

| 类型 | 说明 |
| --- | --- |
| 对话型 | 基于自然语言对话的智能体，适合多轮交互场景。 |
| 流程编排型 | 基于流程编排的智能体，适合复杂业务流程自动化。 |
| 一句话生成 AI | 通过一句话快速生成智能体（敬请期待）。 |

#### 填写基本信息

选择类型后，配置以下信息：

| 配置项 | 说明 |
| --- | --- |
| 智能体名称 | 为 Agent 命名（必填），系统会自动生成默认名称。 |
| 智能体功能介绍 | 简要描述 Agent 的用途和功能。 |

完成基本信息后，进入后续配置页面选择模型、工具和技能等。

## 文件管理

操作 Agent 工作空间中的文件，解决 Agent 与用户之间的文件传递问题。对话中涉及文件时，系统自动引导用户到文件管理页面下载。

| 功能 | 说明 |
| --- | --- |
| 浏览 | 浏览工作空间的目录结构和文件内容。 |
| 上传 | 上传文件到工作空间。 |
| 下载 | 从工作空间下载文件。 |
| 编辑 | 在线编辑工作空间中的文件。 |
| 新建 | 在工作空间中新建文件。 |

## 知识库

创建和管理知识库，实现基于私有数据的智能问答（RAG）。

| 功能 | 说明 |
| --- | --- |
| 创建知识库 | 支持文件上传方式创建知识库。 |
| 外部知识库连接 | 支持连接 RAGFlow 等外部知识库服务。 |
| 文档管理 | 查看、删除、刷新知识库中的文档。 |
| 知识库列表 | 展示知识库名称、标签、最新版本和关联 Agent 数量。 |

## Prompt 管理

**

**说明**

Prompt 管理页面不在侧边栏中直接展示，通过 URL 路由访问。

创建、编辑和管理提示词，提供版本管理和调试能力。

| 功能 | 说明 |
| --- | --- |
| 创建和编辑 | 创建新的 Prompt 或编辑已有 Prompt。 |
| 版本管理 | 跟踪 Prompt 的历史版本。 |
| 调试与对比测试 | 调试功能可直接测试 Prompt 效果，对比测试支持同时运行多个版本在不同模型或参数下的表现，快速定位最优配置。 |
| 提示词变量 | 定义变量名、描述和变量值，方便在不同场景下复用和调优。 |

## 记忆与数据库

管理 Agent 的长期记忆和数据库存储。

- 创建记忆库：配置独立的 LLM 和 Embedding 模型。
- 管理长期记忆和短期记忆。长期记忆持久化存储，跨会话保留；短期记忆仅在当前会话有效，会话结束后自动清除。
- 查看和删除记忆条目。

## 工具与技能

注册和管理 Agent 可调用的外部能力。页面提供五个筛选 Tab：全部、Function Call、MCP、Skill 和组合工具。

| 工具类型 | 说明 |
| --- | --- |
| Function Call | 注册自定义函数调用工具，扩展 Agent 能力。 |
| MCP | 注册 MCP（Model Context Protocol）协议工具。 |
| Skill | 注册技能，支持 Markdown 定义或上传技能包两种创建方式。 |
| 组合工具 | 将多个工具组合为一个复合工具，简化调用流程。 |

## 用户空间

用户空间是开发者进行团队协作的独立工作单元。用户空间通过顶栏的下拉切换器管理，而非侧边栏页面。

| 功能 | 说明 |
| --- | --- |
| 创建空间 | 创建独立的用户空间。 |
| 成员管理 | 查看成员列表、搜索成员、管理角色（管理员或使用者）。 |
| 申请加入 | 申请加入其他用户空间。 |
| 审批 | 空间管理员审批加入申请。 |

## 帮助页面

侧边栏底部提供以下五个独立帮助页面：

- **快速开始**：平台入门指南。
- **使用场景**：典型使用场景和案例。
- **最佳实践**：推荐的使用方式和最佳实践。
- **API 文档**：API 接口文档。
- **常见问题**：常见问题解答（FAQ）。

## 个人资料

通过顶栏用户头像菜单访问，包含以下功能：

- **基本信息**：查看和编辑个人资料。
- **账号安全**：修改登录密码。
