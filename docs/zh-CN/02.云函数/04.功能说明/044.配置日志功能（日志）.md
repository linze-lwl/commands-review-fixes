# 配置日志功能

函数计算支持与日志服务进行集成，为函数配置日志后，函数计算会自动收集日志，并把日志投递到指定日志项目（Project）下的日志库（Logstore）。支持通过函数计算控制台查看单请求日志、实例日志、函数日志，也支持通过日志服务控制台利用日志分析能力对日志进行自定义检索。

## 背景信息

[日志服务](https://help.aliyun.com/zh/sls/what-is-log-service)是针对日志类数据的一站式服务，提供日志数据的采集、存储、查询、分析、可视化、告警等全栈能力。通过日志服务存储函数日志，需要在函数中配置日志项目（Project）和日志库（Logstore），并授予函数访问日志服务的权限。配置完成后，函数日志会打印到指定的日志库中。

## 计费说明

### **计费方式**

函数计算不会为配置日志功能收取额外的费用，日志投递到日志服务，日志服务会收取相关费用。

- 当Logstore的计费模式为按使用功能计费时，函数计算服务将日志推送到日志服务后，日志服务将根据存储空间、读取流量、请求数量、数据加工、数据投递等进行收费。更多信息，请参见[按使用功能计费模式计费项](https://help.aliyun.com/zh/sls/billable-items)。
- 当Logstore的计费模式为按写入数据量计费时，函数计算服务将日志推送到日志服务后，日志服务将根据原始写入数据量等进行收费。更多信息，请参见[按写入数据量计费模式计费项](https://help.aliyun.com/zh/sls/billing-items-in-the-pay-per-data-write-mode)。

如果需要修改Logstore的计费模式，请参见[管理LogStore](https://help.aliyun.com/zh/sls/manage-a-logstore#section-evc-rjx-ndb)。

### **停止计费**

如果需要停止计费，则请先通过函数计算控制台[关闭日志功能](#1f961e5cb38y9)，并前往日志服务控制台[删除Logstore](https://help.aliyun.com/zh/sls/manage-a-logstore#section-ezq-vjx-ndb)或者[删除Project](https://help.aliyun.com/zh/sls/manage-a-project/#entry-m7e-hhf-r4z)。

**

**重要**

删除日志服务Project或Logstore操作不可逆，且删除后将无法再投递日志到日志服务，并删除已有数据，因此请注意备份重要日志数据再执行删除操作。

## 使用函数计算控制台配置日志

### **前提条件**

- 函数计算
  
  - [创建函数](https://help.aliyun.com/zh/functioncompute/fc/user-guide/function-instance-1/)
- **可选：**日志服务SLS
  
  - [创建日志项目和日志库](https://help.aliyun.com/zh/sls/resource-management-overview#task-nn1-y2x-ndb)

### **操作步骤**

1. 登录[函数计算控制台](https://fcnext.console.aliyun.com)，在左侧导航栏，选择**函数管理**>**函数列表**。
2. 在顶部菜单栏，选择地域，然后在**函数列表**页面，单击目标函数。
3. 在函数详情页面，选择**配置**页签，单击**高级配置**右侧的**编辑**，在**高级配置**面板，找到**日志**选项，按照以下步骤完成日志配置，然后单击**部署**。
  
  **
  
  **说明**
  
  - 创建函数时默认开启日志功能，系统自动选择以`serverless-<region_id>`开头的**日志项目**和默认**日志库**。
  - 控制台会自动启用**日志分割规则**、**请求级别指标**和**实例级别指标**，并创建查询日志需要的全部索引。
  
  | **配置项** | **是否必填** | **说明** |
  | --- | --- | --- |
  | **日志功能** | 是 | 是否启用日志服务，取值说明如下：<br>- **启用**：启用后，函数计算会将日志导入到您的日志项目。您可以在函数计算控制台、日志服务控制台查询与检索日志，追踪并定位问题。<br>- **禁用**：不启用日志功能，无法持久化函数日志，无法对问题进行追踪和定位。 |
  | **配置方式** | 是 | 选择**自动配置**或**自定义配置**。如果选择**自定义配置**，**日志项目**和**日志库**必填。 |
  | **日志项目** | 是 | 指定存储函数调用日志的目标日志项目。 |
  | **日志库** | 是 | 指定存储函数调用日志的目标日志库。 |
  | **日志分割规则** | 否 | 是否启用日志分割规则。取值说明如下：<br>- **启用**：启用后，函数计算将按日志分割规则进行切分，切分成多个日志段，并逐条写入日志服务。<br>默认的日志分割规则为`^.{0,2}\d{4}-\d{2}-\d{2}`，即匹配符合`xxxx-xx-xx`格式的日期，其中`x`代表数字。该规则将按照行首是否带有日期进行切分，例如您的日志行首是2023-10-10，则该日志将被认为是一块日志的首行，首行和接下来连续不带日期的日志将被作为一条日志写入到日志服务。<br>- **禁用**：不启用日志分割规则，默认使用`\n`进行日志切分。 |
  | **请求级别指标** | 否 | 是否将请求指标的日志导入日志库，强烈建议启用此功能。<br>取值说明如下：<br>- **启用**：启用后，函数每次调用执行的指标信息如函数执行时间、函数占用内存、是否执行出错、是否出现冷启动、冷启动各个步骤耗时等信息将投递到您在日志配置时选择的Logstore。函数计算的调用日志中可以基于这些指标展示请求列表，您也可以基于指标对请求进行自定义筛选，比如筛选全部错误请求，筛选出现冷启动的请求。<br>- **禁用**：无法查看请求执行的详细信息。 |
  | **实例级别指标** | 否 | 是否启用实例级别指标功能。取值说明如下：<br>- **启用**：启用后，函数实例的指标数据将会投递到您在日志配置时选择的Logstore。您可以查看函数实例维度的性能指标，如实例的CPU、内存、网络IO等指标。函数计算会基于实例级别指标数据提供实例粒度的观测能力，为您提供函数实例端到端的监控排查路径。更多信息，请参见[实例级别指标](https://help.aliyun.com/zh/functioncompute/fc-2-0/user-guide/instance-level-metrics#task-2083444)。<br>- **禁用**：无法查看实例指标的详细信息。 |

## 使用Serverless Devs工具配置日志

### 前提条件

[安装并配置Serverless Devs](https://help.aliyun.com/zh/functioncompute/fc/developer-reference/install-serverless-devs-and-docker#section-0rz-d9z-m1l)

### **操作步骤**

1. 创建代码目录，目录结构如下所示。
  
  ```
  . ├── code │ └── index.js └── s.yaml
  ```
  
  示例中字段logConfig用于配置日志功能，其中logstore和project需修改为您账号下已存在的日志项目和日志库。您也可以直接设置logConfig:auto，函数计算平台将自动指定logstore和project，无需手动配置。
  
  ```
  edition: 3.0.0 name: hello-world-app access: "default" vars: # 全局变量 region: "cn-hangzhou" resources: hello_world: component: fc3 # 组件名称 props: region: ${vars.region} functionName: "start-nodejs-****" description: 'hello world by serverless devs' runtime: "nodejs16" code: ./code handler: index.handler memorySize: 128 timeout: 30 logConfig: # logstore和project需按照实际修改，您也可以使用logConfig:auto，自动创建project和logstore enableInstanceMetrics: true # 启用实例级别指标 enableRequestMetrics: true # 启用请求级别指标 logBeginRule: DefaultRegex # 日志分割规则 logstore: 'your-logstore' # 指定日志库名称（需要提前创建） project: 'your-project' # 指定日志项目名称（需要提前创建）
  ```
2. 执行以下命令部署应用。
  
  ```
  sudo s deploy -y
  ```
  
  执行成功后，您可以登录[函数计算控制台](https://fcnext.console.aliyun.com/)查看函数已部署，以及日志功能已启用。

## **关闭日志功能**

**

**说明**

关闭日志功能后，函数的执行日志将不再被收集和存储，这可能影响问题排查和性能分析，请谨慎操作。

1. 登录[函数计算控制台](https://fcnext.console.aliyun.com)，在左侧导航栏，选择**函数管理**>**函数列表**。
2. 在顶部菜单栏，选择地域，然后在**函数列表**页面，单击目标函数。
3. 在函数详情页面，选择**配置**页签，单击**高级配置**右侧的**编辑**，在**高级配置**面板，找到**日志**选项，将**日志功能**开关置为关闭，然后单击**部署**。

## 查看调用日志

在函数详情页面，单击**日志**页签，查询当前函数的调用记录。

**

**说明**

您可以在函数详情页面，单击**版本管理**或**别名管理**页签，选择要查询的函数版本或别名，单击后进入对应的版本或别名的详情页面，单击**日志**页签查询对应的日志。

**调用请求**页签或**关键词搜索**页签均可以查看函数调用日志的内容。二者的区别：

- **调用请求**
  
  列表显示函数调用结束时间、请求ID、调用结果、实例ID和函数的版本/别名等信息。
  
  - 您可以单击对应的**请求 ID**，查看请求详情和日志详情。
  - 您可以单击对应的**实例 ID**，查看实例的指标使用情况，例如CPU使用情况、内存使用情况和网络流量等。
  - 您可以单击**操作**列的**请求日志**查看日志详情，或单击**操作**列的**高级日志**跳转到[日志服务控制台](https://sls.console.aliyun.com/lognext/profile)查询和分析日志。
- **关键词搜索**
  
  列表显示函数调用时间、日志内容、函数的版本/别名等信息。您可以单击**操作**列的**日志上下文**查找这条日志的若干条上下文日志，或单击**操作**的**日志库**跳转到[日志服务控制台](https://sls.console.aliyun.com/lognext/profile)查询和分析日志。
  
  您可以在搜索框输入关键词查询您所需的日志信息。例如您需要搜索包含文本`hello world`的日志，可以在搜索框输入关键词`hello world`，搜索结果为所有包含该关键词的日志行。更多关于查询关键词的语法，请参见[查询语法与功能](https://help.aliyun.com/zh/sls/query-syntax/#concept-tnd-1jq-zdb)。
