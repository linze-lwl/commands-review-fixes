# 2025年功能发布记录

本文介绍函数计算FC的产品功能和对应的文档动态。

## **2025年12月**

| **功能名称** | **变更类型** | **功能描述** | **相关文档** |
| --- | --- | --- | --- |
| AIO沙箱 (All-in-One Sandbox) | 新增 | AIO沙箱是集成 BrowserTool与Code Interpreter能力的统一云端隔离环境，提供一站式的网页自动化、复杂计算、文件处理与交互式终端能力。 | [AIO沙箱](https://help.aliyun.com/zh/functioncompute/fc/aio-sandbox) |
| GPU混合实例模式<br>（上线地域：深圳、上海） | 新增 | 支持“常驻实例 + 弹性实例”混合实例模式：请求优先由常驻实例承接稳定流量，容量不足时自动启用弹性实例应对峰值。弹性实例空闲时自动进入低成本状态，在保证稳定与可用性的同时降低整体资源成本。 |  |
| GPU全量卡型支持按请求弹性 | 新增 | 所有 GPU 卡型现已全面支持按请求量自动弹缩。系统根据实时请求并发量动态创建和释放实例，实现资源消耗与实际业务负载的匹配，为非持续性任务场景提供更灵活的降本方案。（注：新实例启动存在冷启动耗时，建议时延敏感型业务配置预留实例） |  |
| GPU镜像容量上限提升至30GB | 优化 | 调整 GPU 函数部署时支持的压缩后镜像大小上限，由 15GB 提升至 30GB，提升大型依赖/大模型镜像的部署成功率，减少镜像拆分与多阶段构建成本。 | [配额与使用限制](https://help.aliyun.com/zh/functioncompute/fc/product-overview/limits-of-usage) |
| PolarDB动态挂载能力<br>（上线地域：中国香港、美西、新加坡） | 新增 | 支持 PolarDB 存储目录的动态挂载。允许不同实例间共享持久化数据与大型模型权重文件，在提升数据访问性能的同时，优化了企业级复杂应用的交付与运维效率。 |  |
| OSS动态挂载能力<br>（上线地域：中国香港、美西、新加坡） | 新增 | 支持在函数运行时动态挂载或卸载 OSS Bucket 目录，无需重启服务或重构镜像。该功能实现了计算与大规模非结构化数据的深度解耦，方便模型训练数据加载与多媒体素材流式处理。 |  |
| HTTP触发器原生CORS策略配置 | 新增 | 提供原生 CORS（跨源资源共享）规则配置化能力。支持按需自定义允许来源 (Origin)、请求方法、Header 头部及预检请求缓存时间。无需额外编写中间件代码，即可安全地满足浏览器端跨域调用需求。 |  |
| 控制台概览页升级与 FunctionQ助手 | 优化 | 全新改版函数计算（FC）概览页，包括成本中心、资源管理、解决方案+AI，并上线智能开发助手 FunctionQ，为开发者提供精准的文档索引与技术支持。 |  |
