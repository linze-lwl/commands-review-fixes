# 如何筛选执行时间较长的请求？

长时间的请求会消耗较多的资源，并且影响服务响应速度和整体的稳定性。为了提升系统性能和用户体验，识别并优化执行时间较长的请求非常重要。因此，您需要先开启日志功能，然后通过对SLS日志进行过滤并筛选出执行时间较长的请求，再针对这些请求进行分析优化。

针对SLS日志进行过滤筛选的具体操作为：

1. 在函数详情页面的**日志**页签，单击**高级日志**。
2. 在查询框中输入要过滤的内容，即可查询对应的日志详情。
  
  例如，您可以输入`durationMs > 500`过滤出函数执行时间大于500毫秒的请求。
  
  在函数详情页选择**日志**>**高级日志**页签，日志源选择**function-log**，在查询框中输入查询条件后执行查询。查询结果中可展开日志条目，查看`durationMs`、`isColdStart`、`invocationType`、`functionName`等字段详情，通过`requestId`定位具体请求。
