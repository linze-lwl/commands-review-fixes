# LLM推理模型服务指标监控集成方案

通过在函数实例中以Sidecar模式部署OpenTelemetry Collector，可自动采集并上报LLM推理引擎暴露的Prometheus指标（如Token吞吐量、请求延迟等），无需修改业务代码，即可将指标无缝对接到阿里云ARMS托管Prometheus服务，从而实现对LLM服务的生产级、可视化、可告警的实时监控。

## **准备工作**

开始具体操作前，您需要完成以下必备的云服务与资源的准备工作，并获取后续配置所需的关键信息和权限，如您全部满足，可直接进行后续操作。

### 开通对象存储OSS服务并创建bucket

用于存放模型文件、 OpenTelemetry Collector 的二进制文件及配置文件，以便函数计算实例通过挂载方式访问。如已有可用bucket，可以直接开始进行：

1. 开通[对象存储OSS](https://oss.console.aliyun.com/overview)服务，在左侧导航栏进入**Bucket 列表**页面，单击**创建 Bucket**；
2. 配置以下关键参数，其余保留默认值：
  
  - Bucket名称：输入一个全局唯一的名称，为确保唯一，建议使用项目名-地域-随机字符串的组合，例如`llmfiles-hangzhou-a1b2c3`；
  - 地域：建议选择与在函数计算中部署的模型服务相同地域；
3. 单击**完成创建**。

### 在函数计算中部署LLM推理模型

如果您还没有在函数计算中部署可用的LLM推理模型服务，您可以参考如下步骤，部署一个基于vllm推理引擎部署的Qwen3-0.6B模型进行学习和测试，如已准备好，可以跳过进行下一项：

1. 下载模型文件并上传至OSS，下载地址：[Qwen3-0.6B](https://modelscope.cn/models/Qwen/Qwen3-0.6B/files)；
  
  **
  
  **重要**
  
  模型文件`model.safetensors`较大，请耐心等待所有文件的状态为上传成功。
2. 进入**[函数计算控制台](https://fcnext.console.aliyun.com/overview)**>**函数**，在页面最上方选择地域，点击**创建函数**；
3. 选择**GPU 函数**，点击**创建{title}**；
4. 在**创建函数**页面进行如下核心配置，其余配置保持默认或参考[创建GPU函数](https://help.aliyun.com/zh/functioncompute/fc/user-guide/creating-a-gpu-function/)按需配置：
  
  1. **函数名称**：只能包含字母、数字、下划线和中划线。不能以数字、中划线开头。长度在 1-64 之间，建议使用模型-随机字符串的组合，例如：qwen3-06b-a1b2c3；
  2. **实例类型**：选择**弹性实例**，其余配置保持默认；
  3. **示例代码****：**选择**使用自定义仓库中的镜像**；
  4. **容器镜像**：
    
    ```
    serverless-registry.cn-hangzhou.cr.aliyuncs.com/functionai/vllm-openai:v0.10.1
    ```
  5. **启动命令**：
    
    ```
    vllm serve /mnt/qwen3 --port 9000 --served-model-name Qwen/Qwen3-0.6B --trust-remote-code
    ```
  6. **监听端口**：9000；
  7. **执行超时时间**：建议配置为600；
  8. **权限****：****函数角色**选择AliyunAliyunfcDefaultRole；
  9. **存储**：开启**挂载 OSS 对象存储**，Bucket/Bucket子目录选择模型文件所在子目录，函数本地目录填写`/mnt/qwen3`（与**启动命令**中的目录一致）；
5. 点击**创建**，跳转至**函数详情页**；
6. 点击**函数拓扑图**中的**触发器**，在**编辑触发器 {name}**页面，将**认证方式**变更为无需认证。
  
  **
  
  **说明**
  
  仅测试时需要将**认证方式**变更为无需认证，在生产环境中，建议参考[认证鉴权](https://help.aliyun.com/zh/functioncompute/fc/user-guide/http-triggers-overview#title-zkw-qt1-iz7)进行配置。

2. 在**Prometheus监控**>**实例列表**页面，点击**新建Prometheus实例**；

## **操作步骤**

### **步骤一：**下载并配置 OpenTelemetry Collector

OpenTelemetry Collector 的核心任务是采集 LLM 推理服务暴露的 Prometheus 指标，并通过 Remote Write 协议将其推送到远端的 Prometheus 实例，

1. 下载并解压：[otelcol-contrib_0.136.0_linux_amd64.zip](https://help-static-aliyun-doc.aliyuncs.com/file-manage-files/zh-CN/20251231/bxzpjc/otelcol-contrib_0.136.0_linux_amd64.zip)，解压成功后，将得到名为`otelcol-contrib`的二进制文件和名为`otel_config.yaml`的配置文件；
  
  本文以版本[v0.136.0](https://github.com/open-telemetry/opentelemetry-collector-releases/releases/tag/v0.136.0)为例，其他版本下载请参考：[opentelemetry-collector-releases](https://github.com/open-telemetry/opentelemetry-collector-releases/releases)
2. 修改`otel_config.yaml`配置文件进行如下参数配置，完整的参数说明参考[opentelemetry/configuration](https://opentelemetry.io/docs/collector/configuration/)：
  
  - `**job_name:**`：任务名称，建议与要获取指标的函数名称一致，方便后续进行指标查询和分析；
  - `**targets**`**：**将`{port}`替换为函数的监听端口，如[在函数计算中部署LLM推理模型](#41c3e286024eo)操作中创建的函数，监听端口为9000；
  - 获取方式：**函数管理**>**函数**页面，点击对应**函数名称**，进入**函数详情**页，**配置**>**基础配置**>**监听端口**
  - `**username/password**`**：**配置为阿里云账号的AccessKey_ID/AccessKey_Secret, 如果是RAM用户，需要确保RAM用户有**AliyunPrometheusMetricWriteAccess**或**AliyunCloudMonitorFullAccess**权限，具体可以参考：[通过Remote Write地址将开源Prometheus数据写入阿里云Prometheus实例](https://help.aliyun.com/zh/cms/cloudmonitor-1-0/user-guide/write-open-source-prometheus-data-to-an-alibaba-cloud-prometheus-instance)；
  - `**endpoint**`**：**填写Prometheus实例的公网Remote Write地址；
    
    获取方式：进入[Prometheus实例列表](https://arms.console.aliyun.com/?spm=5176.arms.0.0.4f58f1677yNa01#/prom/cn-shanghai)，点击对应**实例名称/实例Id**，在设置页面中查看并复制。
  - `**scrape_interval**`**（可选）：**指标拉取时间间隔
  
  一个完整的`otel_config.yaml`配置文件示例如下：
  
  ```
  receivers: prometheus: config: scrape_configs: - job_name: "{uner-vllm}" scrape_interval: 15s metrics_path: /metrics static_configs: - targets: ["0.0.0.0:9000"] processors: batch: extensions: basicauth/prw: client_auth: username: "LT******S2K" password: "t6Z7o******B55" exporters: prometheusremotewrite: endpoint: "https://workspace-default-******.cn-hangzhou.******/api/v1/write" auth: authenticator: basicauth/prw service: extensions: [basicauth/prw] pipelines: metrics: receivers: [prometheus] processors: [batch] exporters: [prometheusremotewrite]
  ```

### **步骤二：上传二进制及配置文件**至OSS

1. 在**Bucket 列表**页面，点击对应Bucket名称；
2. 在左侧导航栏进入**文件管理**>**文件列表**，然后单击**上传文件**；
3. 在上传面板中，将[步骤一](#8f3cc8b8899bf)中`otelcol-contrib`二进制文件和修改后的`otel_config.yaml`分别拖拽到上传区域；
4. 其他参数保持默认，单击**上传文件**。

### **步骤三：修改函数实例配置**

1. 进入[函数计算控制台](https://fcnext.console.aliyun.com/overview)，选择**函数管理**>**函数**页面，点击对应**函数名称**；
2. **启动参数配置：**默认情况下**，**LLM指标是打开的，如本文示例的vllm引擎，不需要执行此步骤；有些则是需要通过启动参数显示打开。
  
  比如SGLang，需要点击**配置**>**基础配置**的编辑按钮，在**启动命令**中增加`--enable-metrics`;
3. 编辑**配置**>**高级配置**>**存储**：开启**挂载 OSS 对象存储**，配置**OSS 挂载点**：
  
  **Bucket**/**Bucket 子目录**：选择`otelcol-contrib`二进制文件和修改后的`otel_config.yaml`所在的Bucket及目录；
  
  **
  
  **说明**
  
  如您通过[在函数计算中部署LLM推理模型](#05347119a4qyh)已挂载过OSS对象存储，可通过**添加挂载点**增加一个新的Bucket。
4. **函数本地目录：**填写需要挂载到函数实例本地的目录地址，如`/mnt1/`；
5. 编辑**配置**>**实例配置**：
  
  1. 开启**实例预热**；
  2. 填写**预热超时时间**：10；
    
    执行 Initializer 回调程序的超时时间。单位为秒，取值范围为 1 ～ 300。可以根据实际情况调整。
  3. **预热程序类型**选择**执行指令**，**指令内容**：
    
    **
    
    **说明**
    
    下述内容中的/mnt1需替换为实际挂载的函数本地目录
    
    ```
    nohup /mnt1/otelcol-contrib --config=/mnt1/otel_config.yaml >/dev/null 2>&1 &
    ```
  4. 点击**部署**，完成配置。

curl --request POST \ --url https://{fc_public_endpoint}/v1/chat/completions \ --header 'Authorization: Bearer REPLACE_BEARER_TOKEN' \ --header 'content-type: application/json' \ --data '{"messages":[{"content":"Hello! 你是谁？","role":"user"}],"model":"Qwen/Qwen3-0.6B","stream":false}'

1. 进入[**ARMS控制台**](https://arms.console.aliyun.com/)>**Prometheus 监控**>**实例列表**，点击对应的**Prometheus**实例名称；
