# 使用函数计算搭建前端CICD系统

本文介绍通过函数计算搭建前后端分离架构下的前端静态资源部署服务。

## 背景信息

企业一般使用Gitlab托管前端代码，搭建Jenkins监听Webhook消息，再发布至OSS。此方案虽然比较灵活，但存在以下劣势：

- 需要购买服务器自建Jenkins，涉及计算存储费用。
- 需要专业的运维人员保障服务器稳定运行。
- 集成OSS的SDK实现复杂。

## 方案简介

方案架构如下：

![image](../../../../assets/zh-CN/2513655-001-0018f0ea4b.svg)

方案流程如下：

1. 当您提交代码至代码库时，函数计算会监听前端代码库提交的变更。
2. 函数计算下载提交的分支并上传文件至OSS。
3. 通过CDN进行前端资源加速。

## 方案优势

- 面向Serverless：无需购买服务器。
- 免运维：无需部署配置Jenkins。
- 全方位：提供日志查询、性能监控和报警等功能。
- 一站式：事件驱动方式触发响应。
- 费用低：按需付费。更多信息，请参见[计费概述](https://help.aliyun.com/zh/functioncompute/fc/product-overview/billing-overview-of-fc)。
